**Example Workflow:**

1. **Frontend Dev** wants to build the login page:
`git checkout -b feat/frontend`
2. **ML Dev** wants to add a new LSTM model:
`git checkout -b feat/ml`
3. **DB Dev** wants to connect to Hive:
`git checkout -b feat/db`

**To Save & Sync:**

```bash
git add .
git commit -m "Added [specific feature name]"
git push origin feat/[your-branch-name]

```

*Then, go to GitHub and click **"New Pull Request"**.*

## üé® For the Frontend Team
### üåê Frontend (React/Next.js)

- **Base URL:** `http://127.0.0.1:8000/api/v1/climate`
- **Leaderboard:** Fetch from `/rankings?metric=temp_change`.
- **Comparison:** Fetch from `/compare?regions=Delhi&regions=Mumbai`.
- **Labels:** Use the `color` field from the API to set UI badges (Red/Orange/Green).
* **Backend URL:** `http://127.0.0.1:8000`
* **API Documentation:** `http://127.0.0.1:8000/docs` (Swagger UI)
* **Integration:** Use `axios` or `fetch` to call endpoints.
Endpoints to Use:

GET /rankings?metric=temp_change -> Populates the Leaderboard.

GET /compare?regions=Delhi&regions=Mumbai -> Side-by-side analysis.

GET /severity/{region} -> Returns the color and label for UI badges.

* **Note:** CORS is already enabled on the backend to allow your local dev server to connect.

## ü§ñ For the ML 

* **Workspace:** `backend/ml/`
* **Goal:** We need a prediction function that takes region data and returns temperature/rainfall.
- **Task:** Create models to predict `severity_index` (0-100) and anomaly_flag.
- **Interface:** Export models to `saved_models/` for the Backend to load.
* **Delivery:** Save your trained models in `backend/ml/saved_models/`.
* **Code Interface:** Your inference logic should be placed in `backend/app/services/forecast_service.py`.

---

## üóÑÔ∏è For the Database

* **Workspace:** `backend/database/`
* **Goal:** Connect FastAPI to the Hadoop cluster using **PyHive** or **HappyBase**.
* **Delivery:** Update `backend/database/db.py` with the connection string.
* **Schema:** Focus on the `historical_weather` table in Hive/HDFS.

## üóÑÔ∏è For everyone

üóÑÔ∏è Database Schema Definition1. Table: historical_weatherStored in Hadoop HDFS / Hive. This is the source for ML training.
Column,Type,Description
id,UUID,Unique record identifier
timestamp,Timestamp,Date and time of the weather reading
region_id,Integer,Foreign key linking to the Region table
temp_avg,Float,Average temperature recorded
precipitation,Float,Total rainfall in mm
humidity,Float,Humidity percentage (0-100)

2. Table: climate_outputsThis table stores the results generated by our ML models.
Column,Type,Description
region_id,Integer,Region code identification
prediction_date,Date,The future date this prediction is for
severity_index,Float,Calculated risk score (0.0 - 100.0)
anomaly_flag,Boolean,True if the data deviates significantly from norms
